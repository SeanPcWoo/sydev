  <action>
更新 `packages/core/src/api-routes.ts`，实现模板相关端点（替换 Plan 01 的 501 placeholder）：

1. `GET /api/templates` — 查询参数 `?type=workspace|project|device|full` 可选过滤。调用 templateManager.list(type?)，返回 TemplateMeta 数组。

2. `POST /api/templates` — body: `{ name, description, type, data }`。调用 templateManager.save(name, description, type, data)，返回 201 + TemplateMeta。验证失败返回 400。

3. `GET /api/templates/:id` — 调用 templateManager.load(id)，返回 `{ meta, content }`。不存在返回 404 + `{ error: "模板不存在" }`。

4. `PUT /api/templates/:id` — body: `{ name, description, type, data }`。先检查 exists(id)，不存在返回 404。存在则调用 save() 覆盖（save 已支持覆盖同 ID），返回 200 + TemplateMeta。

5. `DELETE /api/templates/:id` — 调用 templateManager.delete(id)，返回 204。不存在返回 404。

6. `POST /api/templates/import` — body: 完整的模板 JSON 内容（`{ type, data }` 格式）+ `{ name, description }` 元数据。用 templateContentSchema 验证 content，验证通过后调用 save()，返回 201。

7. `GET /api/templates/:id/export` — 调用 templateManager.load(id)，返回完整 JSON（meta + content），设置 Content-Disposition header 为 `attachment; filename="{id}.json"` 触发下载。

TemplateManager 实例在 registerApiRoutes 中创建，baseDir 使用 cwd（通过参数或 app.locals 传入）。
