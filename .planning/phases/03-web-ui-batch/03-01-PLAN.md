  <action>
1. 在 packages/core 中安装 express 和 ws 依赖（`pnpm --filter @sydev/core add express ws` + `pnpm --filter @sydev/core add -D @types/express @types/ws`）。使用 Express 作为 HTTP 框架（成熟稳定，TypeScript 支持好），ws 作为 WebSocket 库（轻量原生）。

2. 创建 `packages/core/src/web-server.ts`：
   - 导出 `createWebServer(options: { port?: number; staticDir?: string; open?: boolean })` 函数
   - 创建 Express app，挂载 `express.json()` 中间件
   - 如果 staticDir 存在，用 `express.static(staticDir)` 提供前端静态文件，所有非 /api 路由 fallback 到 index.html（SPA 支持）
   - 调用 `registerApiRoutes(app)` 注册 API 路由
   - 创建 HTTP server，在其上创建 WebSocket server（ws 库）
   - 创建 WsBridge 实例连接 WebSocket 和 ProgressReporter
   - 监听端口，启动后如果 open=true 则用 `import('open')` 打开浏览器（lazy import，避免硬依赖）
   - 返回 `{ server, app, wsBridge, close() }` 对象

3. 创建 `packages/core/src/api-routes.ts`：
   - 导出 `registerApiRoutes(app: Express)` 函数
   - 注册以下 REST 端点骨架（后续 plan 填充实现）：
     - `GET /api/status` — 返回 `{ ok: true }` 健康检查
     - `POST /api/workspace/init` — placeholder 返回 501
     - `POST /api/project/create` — placeholder 返回 501
     - `POST /api/device/add` — placeholder 返回 501
     - `GET /api/templates` — placeholder 返回 501
     - `POST /api/templates` — placeholder 返回 501
     - `GET /api/templates/:id` — placeholder 返回 501
     - `PUT /api/templates/:id` — placeholder 返回 501
     - `DELETE /api/templates/:id` — placeholder 返回 501
     - `POST /api/templates/import` — placeholder 返回 501
     - `GET /api/templates/:id/export` — placeholder 返回 501
     - `POST /api/batch/execute` — placeholder 返回 501
   - 全局错误处理中间件，返回 `{ error: string }` JSON

4. 创建 `packages/core/src/ws-bridge.ts`：
   - 导出 `WsBridge` 类
   - 构造函数接收 `WebSocketServer` 实例
   - `attachReporter(reporter: ProgressReporter)` 方法：监听 reporter 的 step/success/error/output 事件，JSON 序列化后广播给所有连接的 WebSocket 客户端
   - `broadcast(type: string, data: unknown)` 方法：遍历 wss.clients 发送 `{ type, data, timestamp }` JSON
   - `detachReporter()` 方法：移除所有事件监听

5. 更新 `packages/core/src/index.ts`：添加 web-server、api-routes、ws-bridge 的导出。

6. 创建 `apps/cli/src/commands/web.ts`：
   - 导出 `webCommand`（Commander Command）
   - 命令名 `web`，描述 "启动 Web 可视化界面"
   - 选项：`-p, --port <port>` 端口号（默认 3456），`--no-open` 不自动打开浏览器
   - action：调用 `createWebServer({ port, open })`，打印启动信息（`chalk.green` 显示 URL）
   - 注册 SIGINT 处理器优雅关闭

7. 在 `apps/cli/src/index.ts` 中注册 webCommand，并在 preAction hook 中跳过 web 命令的环境检查（Web 服务不需要 RealEvo-Stream 环境）。

注意：开发阶段 staticDir 可以不传（API-only 模式），前端通过 Vite dev server 代理到后端。生产模式下 staticDir 指向 apps/web/dist。
