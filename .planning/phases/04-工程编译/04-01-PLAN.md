```typescript
export interface ScannedProject {
  name: string;       // 目录名
  path: string;       // 绝对路径
}

export class WorkspaceScanner {
  constructor(private workspaceRoot: string) {}

  // 扫描并返回有效工程列表
  scan(): ScannedProject[]
}
```

**扫描逻辑：**
1. 读取 `workspaceRoot` 的直接子目录（`fs.readdirSync` + `dirent.isDirectory()`）
2. 跳过以 `.` 开头的隐藏目录
3. 跳过已知构建输出目录：`['Debug', 'Release', 'debug', 'release', 'node_modules', 'dist']`
4. 检查目录内同时存在 `.project` 文件 **和** `Makefile` 文件（`fs.existsSync`）
5. 两者都存在才加入结果列表

不抛异常：如果 `workspaceRoot` 不存在或读取失败，返回空数组。

创建 `BuildRunner` 类，封装 bear/make 子进程执行、WORKSPACE_ 环境变量注入、输出流处理和错误摘要提取。

**接口定义：**
```typescript
export interface BuildOptions {
  extraArgs?: string[];    // -- 之后的自定义 make 参数
  quiet?: boolean;         // 单工程静默模式（不透传 stdout）
  verbose?: boolean;       // 批量模式详细输出（透传每个工程 stdout）
}

export interface BuildProjectResult {
  name: string;
  success: boolean;
  durationMs: number;
  stdout: string;          // 完整 stdout（即使静默也保留供汇总）
  stderr: string;
  errorSummary?: string;   // 失败时：首条含 "error:" 的行（不区分大小写）
  errorLines?: string[];   // 失败时：所有含 "error:" 的行（最多 10 条）
}

export interface BuildResult {
  projects: BuildProjectResult[];
  total: number;
  succeeded: number;
  failed: number;
}

export type BuildProgressEvent = {
  type: 'project-start';
  name: string;
  index: number;
  total: number;
} | {
  type: 'project-done';
  result: BuildProjectResult;
  index: number;
  total: number;
} | {
  type: 'stdout-line';
  name: string;
  line: string;
};

export class BuildRunner extends EventEmitter {
  constructor(private projects: ScannedProject[]) {}

  // 检测 bear 是否可用（which bear）
  private async checkBear(): Promise<boolean>

  // 构建所有 WORKSPACE_<NAME> 环境变量（所有已扫描工程）
  private buildEnv(): NodeJS.ProcessEnv

  // 编译单个工程，emit 'progress' 事件（BuildProgressEvent）
  async buildOne(
    project: ScannedProject,
    options?: BuildOptions
  ): Promise<BuildProjectResult>

  // 批量编译所有工程（顺序执行），emit 'progress' 事件
  async buildAll(options?: BuildOptions): Promise<BuildResult>
}
```

**实现要点：**

`checkBear()`：执行 `which bear`，成功（exit 0）返回 true，否则 false。

`buildEnv()`：以 `{ ...process.env }` 为基础，为每个 project 添加：
`WORKSPACE_${project.name.toUpperCase().replace(/-/g, '_')}=project.path`

`buildOne(project, options)`：
1. 调用 `checkBear()`，若可用使用 `['bear', '--', 'make']`，否则 `['make']`（并 emit 降级提示 warning event）
2. 追加 `options.extraArgs`（如有）
3. 用 `spawn(cmd, args, { cwd: project.path, env: buildEnv() })`
4. 收集 stdout/stderr 字符串，同时 emit `{ type: 'stdout-line', name, line }` 事件（非 quiet 模式下）
5. 记录开始时间，结束时记录 durationMs
6. 失败时提取 errorSummary（stdout + stderr 中第一条含 `error:` 的行），errorLines（所有含 error: 行，最多 10 条）
7. 返回 BuildProjectResult

`buildAll(options)`：
1. 顺序遍历 projects
2. 每个工程前 emit `{ type: 'project-start', name, index, total }`
3. 调用 `buildOne(project, options)`（批量时默认 quiet=true，除非 verbose=true）
4. 完成后 emit `{ type: 'project-done', result, index, total }`
5. 返回聚合 BuildResult

**Ctrl+C 处理：** `buildOne` 内监听 `process.on('SIGINT')`，收到后 kill 子进程，向调用者 reject（让 CLI 层处理）。

注意：不要 import `executeRlCommand`，BuildRunner 直接使用 `child_process.spawn`（需要流式输出）。
