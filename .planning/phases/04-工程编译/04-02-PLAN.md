```typescript
export interface BuildOptions {
  extraArgs?: string[];
  quiet?: boolean;
  verbose?: boolean;
}

export interface BuildProjectResult {
  name: string;
  success: boolean;
  durationMs: number;
  stdout: string;
  stderr: string;
  errorSummary?: string;
  errorLines?: string[];
}

export interface BuildResult {
  projects: BuildProjectResult[];
  total: number;
  succeeded: number;
  failed: number;
}

export type BuildProgressEvent =
  | { type: 'project-start'; name: string; index: number; total: number }
  | { type: 'project-done'; result: BuildProjectResult; index: number; total: number }
  | { type: 'stdout-line'; name: string; line: string };

export class BuildRunner extends EventEmitter {
  constructor(projects: ScannedProject[]) {}
  async buildOne(project: ScannedProject, options?: BuildOptions): Promise<BuildProjectResult>
  async buildAll(options?: BuildOptions): Promise<BuildResult>
}
```

创建 `apps/cli/src/commands/build.ts`，实现以下三种执行路径：

**命令结构：**
```
sydev build [工程名] [--all] [--quiet] [--verbose] [-- <make参数>]
```

子命令 `sydev build init` 在 Task 2 中处理，用 `.command('init')` 挂在同一 buildCommand 下。

**1. 无参数 — 交互式选择：**
- 用 WorkspaceScanner 扫描 `process.cwd()`
- 若无工程：打印黄色提示"未找到工程（确认当前目录是 workspace 根目录，且工程子目录同时包含 .project 和 Makefile）"后 exit 1
- 用 Inquirer `checkbox` 让用户从列表中选择一个或多个工程（显示工程名）
- 用户选择后，逐个调用 `BuildRunner.buildOne()`（透传模式），按顺序编译所选工程
- 每个工程编译前打印 `chalk.cyan('编译 ${name}...')`
- 失败时打印完整错误摘要（见下方错误摘要格式），最终 exit code = 失败工程数

**2. 有工程名参数：**
- 用 WorkspaceScanner 找到匹配工程（名字精确匹配）
- 未找到则打印错误"未找到工程 'xxx'，运行 sydev build 查看可用工程列表"后 exit 1
- 调用 `BuildRunner.buildOne(project, { quiet: opts.quiet, extraArgs })`
- 监听 `runner.on('progress', event)` 中的 `stdout-line` 事件，直接 `process.stdout.write(line + '\n')`
- 监听 bear 降级 warning：BuildRunner 若 emit `{ type: 'warning', message }` 则打印 `chalk.yellow('⚠ ' + message)`
- 成功：打印 `chalk.green('✓ 编译成功')` + 耗时
- 失败：打印完整 stdout（如果 quiet 才需要，非 quiet 已实时输出），再打印红色错误摘要，exit 1

**错误摘要格式（单工程失败）：**
```
chalk.red('✗ 编译失败') + chalk.dim(` (${errorLines.length} 个错误)`)
每条 errorLine: chalk.red('  > ' + line)
```

**3. --all 批量：**
- WorkspaceScanner 扫描，无工程则提示 exit 1
- 调用 `BuildRunner.buildAll({ quiet: !opts.verbose, verbose: opts.verbose, extraArgs })`
- 监听 `project-start` 事件：打印进度行 `chalk.dim('[N/Total]') + ' ' + name + chalk.dim(' 编译中...')`（覆盖式更新：用 `process.stdout.write('\r' + line)`）
- 监听 `project-done` 事件：打印完整状态行（覆盖 + 换行）
  - 成功：`chalk.dim('[N/Total]') + ' ' + name + ' ' + chalk.green('✓') + chalk.dim(` ${(durationMs/1000).toFixed(1)}s`)`
  - 失败：`chalk.dim('[N/Total]') + ' ' + name + ' ' + chalk.red('✗') + chalk.dim(` ${time}s`) + chalk.red(' ' + errorSummary)`
- 若 verbose，监听 `stdout-line` 透传输出
- 全部完成后打印汇总表格（见下方汇总格式）
- exit code = result.failed

**批量汇总表格格式：**
```
─────────────────────────────────────────
  工程名          状态   耗时    备注
─────────────────────────────────────────
  libcpu          ✓      2.3s
  bsp-rk3568      ✗      5.1s   error: undefined reference to...
─────────────────────────────────────────
  总计: 2 个工程，1 成功，1 失败
```
用 chalk 对齐，成功行绿色，失败行红色，用 `padEnd` 对齐列宽（工程名列 20 字符，状态 6，耗时 8）。

**Ctrl+C 处理：** 单工程：进程直接退出（由 BuildRunner 处理）。批量：捕获 SIGINT，询问"是否继续编译剩余工程？"（Inquirer confirm）；用户选 No 则 exit，选 Yes 则继续。

**-- 参数透传：** Commander `allowUnknownOption()` + 手动解析 `process.argv` 中 `--` 之后的内容作为 `extraArgs`。

**preAction 跳过：** build 命令不依赖 rl，需跳过全局环境检查。在 `index.ts` 中处理（Task 2）。

**Makefile 生成逻辑：**
1. 用 WorkspaceScanner 扫描 `process.cwd()`，无工程则提示 exit 1
2. 构建 Makefile 内容字符串：

```makefile
# SylixOS Workspace Makefile
# 由 sydev build init 自动生成 — 可直接使用 make 编译，无需 sydev
# 如需更新（新增/删除工程），重新运行 sydev build init

# Workspace 工程路径（自动生成，请勿手动修改此区域）
export WORKSPACE_LIBCPU = /absolute/path/to/libcpu
export WORKSPACE_BSP_RK3568 = /absolute/path/to/bsp-rk3568
# ... 每个工程一行

# ─── 工程 Targets ───────────────────────────────────────────────

# libcpu
libcpu:
	bear -- make -C $(WORKSPACE_LIBCPU)

clean-libcpu:
	$(MAKE) -C $(WORKSPACE_LIBCPU) clean

rebuild-libcpu: clean-libcpu libcpu

cp-libcpu:
	# TODO: 配置产物复制路径
	# cp $(WORKSPACE_LIBCPU)/Debug/libcpu.so /path/to/destination

# bsp-rk3568
bsp-rk3568:
	bear -- make -C $(WORKSPACE_BSP_RK3568)

# ... 每个工程重复上述结构

.PHONY: libcpu clean-libcpu rebuild-libcpu cp-libcpu \
        bsp-rk3568 clean-bsp-rk3568 rebuild-bsp-rk3568 cp-bsp-rk3568
```

规则：
- WORKSPACE_ 变量名：工程名大写，`-` 替换为 `_`
- 值：工程绝对路径（`project.path`）
- 每个工程生成四个 target：`<name>`、`clean-<name>`、`rebuild-<name>`、`cp-<name>`
- `<name>` target 使用 `bear -- make -C $(WORKSPACE_X)`（不检测 bear 是否可用，用户 Makefile 场景下自行处理）
- `cp-<name>` 内只有注释模板，不生成实际 cp 命令
- `.PHONY` 行列出所有 target
- Makefile 使用 TAB 缩进（`\t`），确保生成正确

3. 写入 `path.join(process.cwd(), 'Makefile')`（用 `fs.writeFileSync`）
4. 若文件已存在，先打印"更新已有 Makefile"（不提示覆盖确认，直接更新）
5. 打印成功信息：`chalk.green('✓ Makefile 已生成: ./Makefile')` + 提示"包含 N 个工程 target，可直接运行 make <工程名> 编译"
