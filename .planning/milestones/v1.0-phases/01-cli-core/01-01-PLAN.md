    - Test 3: checkToolchain() 检查 RealEvo-Stream 工具链路径是否存在

  </behavior>
  <action>
在 packages/core/src/ 创建环境检查器模块：

1. types.ts 定义核心类型：
```typescript
export interface RlVersion {
  major: number;
  minor: number;
  patch: number;
  raw: string;
}

export interface EnvCheckResult {
  available: boolean;
  version?: RlVersion;
  error?: string;
  fixSuggestion?: string;
}

export interface ToolchainCheckResult {
  installed: boolean;
  path?: string;
  error?: string;
}

export interface EnvironmentStatus {
  rl: EnvCheckResult;
  toolchain: ToolchainCheckResult;
  overall: boolean;
}
```

2. env-checker.ts 实现检查逻辑：
   - checkRlCommand(): 使用 child_process.execSync 执行 `rl --version`，解析输出提取版本号（正则匹配 \d+\.\d+\.\d+），捕获 ENOENT 错误返回 not found
   - checkToolchain(): 检查环境变量 REALEVO_HOME 或常见安装路径（/opt/realevo, ~/realevo），使用 fs.existsSync 验证
   - checkEnvironment(): 调用上述两个函数，综合返回状态，生成修复建议（如 "请安装 RealEvo-Stream 工具链，访问 https://..."）
   - 所有函数使用 try-catch 包裹，错误信息清晰且可操作

3. env-checker.test.ts 使用 Vitest：
   - Mock child_process.execSync 测试不同场景（命令不存在、版本解析、执行错误）
   - Mock fs.existsSync 测试工具链检查
   - 验证错误信息和修复建议的清晰度

使用 TDD 方式：先写测试，运行失败（RED），再实现功能通过测试（GREEN）。
