1. 创建 `packages/core/src/config-reader.ts`：
   - 导出 `ConfigReader` 类，构造函数接收 `cwd?: string`（默认 process.cwd()）
   - `getWorkspaceStatus(): { configured: boolean; config?: WorkspaceConfig; path: string }`
     - 读取 `{cwd}/.realevo/workspace.json`（RealEvo-Stream workspace 配置文件），如果存在则解析并返回
     - 不存在则返回 `{ configured: false, path: cwd }`
   - `getProjects(): ProjectConfig[]`
     - 读取 `{cwd}/.realevo/projects.json`，解析返回项目列表
     - 不存在则返回空数组
   - `getDevices(): DeviceConfig[]`
     - 读取 `{cwd}/.sydev/devices.json`（Phase 1 建立的设备存储路径），解析返回设备列表
     - 不存在则返回空数组
   - 所有读取操作用 try/catch 包裹，文件不存在或解析失败返回默认空值
   - 注意：这里读取的是实际文件系统状态，不做 schema 验证（状态展示不应因格式问题报错）

2. 更新 `packages/core/src/index.ts`：添加 ConfigReader 导出。

3. 更新 `packages/core/src/api-routes.ts`，添加状态端点：
   - `GET /api/status/workspace` — 调用 configReader.getWorkspaceStatus()，返回 JSON
   - `GET /api/status/projects` — 调用 configReader.getProjects()，返回 JSON 数组
   - `GET /api/status/devices` — 调用 configReader.getDevices()，返回 JSON 数组
   - ConfigReader 实例在 registerApiRoutes 中创建（或通过 app.locals 传入 cwd）
