1. 创建 `packages/core/src/batch-executor.ts`：
   - 导出 `BatchItemStatus` 类型：`{ name: string; type: 'project' | 'device'; status: 'pending' | 'running' | 'success' | 'failed'; error?: string }`
   - 导出 `BatchResult` 类型：`{ total: number; succeeded: number; failed: number; items: BatchItemStatus[] }`
   - 导出 `BatchExecutor` 类：
     - 构造函数接收 `RlWrapper` 和 `ProgressReporter`
     - `async execute(cwd: string, projects: ProjectConfig[], devices: DeviceConfig[]): Promise<BatchResult>`
       - 构建 items 列表（所有 project + device，初始 status=pending）
       - 顺序执行（不并行，避免竞态条件，per CONTEXT.md 决策）
       - 每个 item 执行前：设 status=running，reportStep(`batch:${type}:${name}`, 0)
       - 执行 project：调用 rlWrapper.createProject({ ...project, cwd })
       - 执行 device：调用 rlWrapper.addDevice({ ...device, cwd })
       - 成功：设 status=success，reportStep(`batch:${type}:${name}`, 100)
       - 失败：设 status=failed + error，reportError，继续执行下一个（跳过失败项）
       - 全部完成后返回 BatchResult 汇总
     - `async retryFailed(cwd: string, failedItems: Array<{ type, config }>): Promise<BatchResult>`
       - 只重试指定的失败项，逻辑同 execute

2. 更新 `packages/core/src/index.ts`：添加 BatchExecutor、BatchItemStatus、BatchResult 导出。

3. 更新 `packages/core/src/api-routes.ts`，实现批量端点：
   - `POST /api/batch/execute` — body: `{ cwd: string, projects: ProjectConfig[], devices: DeviceConfig[] }`
     - 用 zod 验证 projects 数组（每项用 projectSchema）和 devices 数组（每项用 deviceSchema）
     - 验证失败返回 400
     - 创建 BatchExecutor，附加 ProgressReporter 到 WsBridge（实时推送进度到前端）
     - 调用 executor.execute()，返回 BatchResult
   - `POST /api/batch/retry` — body: `{ cwd: string, items: Array<{ type: 'project'|'device', config: unknown }> }`
     - 调用 executor.retryFailed()，返回 BatchResult
