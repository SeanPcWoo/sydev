  <action>
    1. 创建 `apps/cli/src/commands/template.ts` 导出 `templateCommand` (commander Command):
       - **template save**: inquirer 提示输入 name、description、type（workspace/project/device/full）。根据 type 用 inquirer 收集配置字段（workspace 字段复用 workspace-schema 定义，project/device 同理，full 依次收集 workspace + 多个 project + 多个 device）。检查 TemplateManager.exists(slugify(name))，冲突时 inquirer 提示：覆盖/重命名/取消。调用 templateManager.save()，chalk.green 显示成功。
       - **template list**: 选项 `-t, --type <type>` 过滤。调用 templateManager.list(type)。无模板时提示，有模板时格式化表格输出 ID、名称、类型、更新时间。
       - **template apply <id>**: 调用 templateManager.load(id)。full 模板用 inquirer checkbox 选择部分应用（workspace、各 project、各 device）。检测冲突时 inquirer 提示：覆盖/合并（模板值优先）/取消。构造 FullConfig 传给 InitOrchestrator.execute()。ora spinner 监听 ProgressReporter 事件显示进度。
       - **template delete <id>**: inquirer confirm 确认，调用 templateManager.delete(id)，显示成功。
       - **template export**: 选项 `-o, --output <file>` 默认 sydev-config.json。inquirer 选择导出类型，收集配置，ConfigManager.exportToJson() 写入文件。
       - **template import <file>**: 读取文件，ConfigManager.importFromJson(fullConfigSchema, json) 验证。失败显示错误，成功后询问是否保存为模板。
       - 所有子命令用 process.cwd() 作为 TemplateManager baseDir。

    2. 创建 `apps/cli/src/commands/init.ts` 导出 `initCommand`:
       - 命令 `init`，选项 `--config <file>`。读取并 JSON.parse 配置文件。创建 ProgressReporter（监听 step/success/error 事件用 ora+chalk 显示）、RlWrapper、InitOrchestrator。调用 orchestrator.execute(config)。成功显示 completedSteps 摘要（chalk.green），失败显示已完成步骤 + 失败步骤和错误（chalk.red）+ 修复建议。

    3. 更新 `apps/cli/src/index.ts`: 导入并 addCommand templateCommand 和 initCommand。在 preAction hook 中为 template（除 apply 外）和 init 跳过环境检查。
